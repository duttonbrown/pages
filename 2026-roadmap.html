Rules of the road

  • Anything in the backlog is explicitly not committed — adding it here is not a promise to do it.
  • Each item carries a "why it matters" line so priority can be re-evaluated later without re-thinking context.
  • "Blocked by" captures dependencies so nothing gets promoted prematurely. Red = hard dependency. Amber = waiting on timing or data.
  • During quarterly planning, review this section first — promote items before adding anything brand new.
How to Use This Document

Weekly

1:1s

Each DRI reviews their row in the Focus Grid and their projects in the detail table. Is the focus still right? Any blockers?

Monthly

Review

Thomas reviews the full grid, updates statuses, and adjusts the following month's focus cells as needed.

Quarterly

Planning

Open the Backlog first — promote items into the grid before planning anything new. Then set the next quarter's theme and cells.

When a team finishes early

Promote from Backlog

Check the Backlog for a promotable unblocked item before inventing new work. Pull the highest-priority item with no red dependency.
